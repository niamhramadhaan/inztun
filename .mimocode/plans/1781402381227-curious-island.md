# Top Bar + Homepage — Implementation Plan

## Overview

Add a persistent top navigation bar with module tabs, global search, and quick access. Create a personalized Homepage module showing favorite tools from all modules without categories.

## Files to Create

### 1. `src/components/TopBar.ts`
Persistent top bar component:
- **Module tabs**: Home (★), Workers Suite, Playground, Design Studio, Marketing Lab, Freelance Core
- **Global search**: Input field that opens CommandPalette on focus/type
- **Quick access**: Row of recently used + favorited tools (from IndexedDB)
- Active tab highlighted based on current route
- Sticky position at top, z-index above workspace

### 2. `src/modules/home/index.ts`
Homepage module:
- Collects favorites from all modules via IndexedDB
- Renders a flat bento grid of favorite tool cards (no categories)
- Shows "Getting Started" section if no favorites yet
- Each card navigates to the tool in its respective module
- `moduleId = 'home'`

## Files to Modify

### 3. `index.html`
Add top bar container:
```html
<div id="app">
  <canvas id="cosmos"></canvas>
  <header id="topbar"></header>
  <main id="workspace"></main>
</div>
```

### 4. `src/styles/base.css`
Add top bar layout styles:
- `#topbar` — fixed top, full width, z-index: 100
- `#workspace` — add `padding-top` to account for top bar height

### 5. `src/main.ts`
- Import and create `TopBar` component
- Register `Home` module as default
- Pass module list to TopBar
- Change default route from `workers-suite` to `home`

### 6. `src/components/CommandPalette.ts`
- Add `nav:home` entry for Home module

## Homepage Module

- Title: "Welcome to inztun"
- Subtitle: "Your personalized workspace"
- Grid of favorite tool cards from all modules
- Each card shows: tool name, icon, module name badge
- Click navigates to `#/{module}/{tool}`
- If no favorites: shows prompt to star tools
